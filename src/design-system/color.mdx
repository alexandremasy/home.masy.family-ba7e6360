## Semantics — what you actually use

Meaning before hue. **These are the only colour tokens a component may reach for.** Each one
points at a rung, never at a hex, so re-pointing `--primary` moves the whole product at once.

### Which rung each semantic points at

| Token                                  | Rung      |
| -------------------------------------- | --------- |
| `--background`                         | Light/00  |
| `--card` · `--popover` · `--sidebar`   | White     |
| `--secondary` · `--muted` · `--accent` | Light/10  |
| `--border` · `--input`                 | Light/20  |
| `--muted-foreground`                   | Light/60  |
| `--foreground` · every `-foreground`   | Light/90  |
| `--primary` · `--ring`                 | Teal/60   |
| `--warm`                               | Orange/40 |
| `--mustard`                            | Yellow/50 |
| `--success`                            | Green/50  |
| `--destructive`                        | Red/50    |

Two hues carry two different jobs, and the distance between them is what makes both readable:
**`--warm` (Orange/40) means "act on this"** — a reading to enter, a level running low.
**`--destructive` (Red/50) means "something failed or will be lost"** — an error, a deletion.

## The same semantics, dark

The dark theme re-points every semantic. Surfaces and text swap; the signal hues stay
recognisable, lifted in lightness so they hold on a dark ground.

## Primitives — the nine ramps

Ported verbatim from the Figma library (_alexandremasy — tokens_, page "Colors"). Each ramp runs
**00** (lightest) to **90** (darkest); 40–60 is the usable band on a light ground.

## The `-foreground` pairing rule

A `-foreground` token only works on its own **solid** fill. Every accent foreground is Light/90
(dark) in both themes, chosen for contrast — warm 6.4:1, mustard 9.0:1, teal 5.1:1.

On a _tint_ such as `bg-warm/15` the fill is nearly the page, so reach for `text-warm`, not the
foreground token.

## Known debt — the dark theme

The Figma library has a single neutral ramp, "Light", and no dark counterpart, so `.dark` is the
only block in `styles.css` that is **authored rather than ported**. It stays navy-tinted rather
than a mechanically inverted Light ramp — inverting was declined on 2026-07-16 because it costs
the dark theme its temperature.

Closing this means adding a Dark ramp in Figma; until then, the dark values point at no rung.
