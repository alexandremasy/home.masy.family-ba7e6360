## The typeface

The **native OS stack** — San Francisco on Apple, Segoe UI on Windows, Roboto on Android. No
webfont is loaded, so there is nothing to wait for and nothing to fall back to.

## The scale

Eleven rungs, nothing else: `styles.css` wipes Tailwind's own scale (`--text-*: initial`) before
declaring these, so `text-7xl` and `text-[13px]` are unwritable.

The names are Tailwind's, the values are Figma's, and they disagree by one rung — `text-xl` is
**24px** here, not Tailwind's 20. Read the value, not the name.

<Typeset
  fontSizes={[10, 12, 14, 16, 20, 24, 28, 32, 40, 48, 56]}
  sampleText="Reads at arm's length"
  fontWeight={400}

| rung        | px  | role                                         |
| ----------- | --- | -------------------------------------------- |
| `text-2xs`  | 10  | the eyebrow, uppercase only                  |
| `text-xs`   | 12  | the body floor — metadata, figures, captions |
| `text-sm`   | 14  | body, the default                            |
| `text-base` | 16  | body, roomy                                  |
| `text-lg`   | 20  | the name of a thing                          |
| `text-xl`   | 24  | section title                                |
| `text-2xl`  | 28  |                                              |
| `text-3xl`  | 32  | page title                                   |
| `text-4xl`  | 40  | the big number                               |
| `text-5xl`  | 48  |                                              |
| `text-6xl`  | 56  | display ceiling                              |

## Weights

Two, and no more. Regular carries running text; semibold carries every heading, value and label
that needs to come forward.

## Eyebrow tracking

The one letter-spacing this system owns: `--tracking-eyebrow`, **0.18em**. Figma sets
letter-spacing 0 on every token, but the eyebrow is a dashboard pattern the reference doesn't
cover, so the app owns its spacing. One value, where there used to be seven.

It belongs to `components/eyebrow.tsx` — hand-spacing an uppercase label means you skipped the
component.

<Unstyled>
  <p className="py-2 text-2xs font-medium uppercase tracking-eyebrow text-foreground">
    Understand · align · direct
  </p>
</Unstyled>

## Known debt — `.font-serif`

`--font-serif` resolves to the **same stack** as `--font-sans`. It is a display _role_ marking
intent at ~145 call sites, and it currently renders no differently. Two ways out, both open:
give it a real display face, or replace it with the weight and tracking it actually stands for.
Until then, do not read `.font-serif` as "a second typeface" — there is only one.
