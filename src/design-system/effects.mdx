## Which one

| Effect                                                             | What it is                                      | Where it runs                         |
| ------------------------------------------------------------------ | ----------------------------------------------- | ------------------------------------- |
| [Ambient Mesh](?path=/docs/foundations-effects-ambient-mesh--docs) | Blurred blobs drifting behind a whole view      | `_app.tsx`, every non full-bleed page |
| [Ambient Glow](?path=/docs/foundations-effects-ambient-glow--docs) | A tinted wash anchored to the top edge of a box | Repas, behind the title band          |
| [Media Sweep](?path=/docs/foundations-effects-media-sweep--docs)   | A gradient drifting under a pane's content      | Room pages, the now-playing card      |

## The contract

Each effect is one class in `styles.css` (`.fx-mesh`, `.fx-glow`, `.fx-sweep`) whose look is
entirely expressed as custom properties. The React wrapper in `src/components/effects/` writes
those properties from props and does nothing else — a prop you leave out keeps the CSS default,
it never blanks it.

So there are two ways to consume an effect, and both are first class:

```tsx
// From React, typed:
<AmbientGlow color="#7c3aed" strength={32} className="absolute inset-x-0 top-0 h-96" />
```

```css
/* Or straight from CSS, when the effect belongs to a rule you already own: */
.my-surface {
  --fx-glow-color: var(--warm);
  --fx-glow-speed: 22s;
}
```

Adding a fourth effect means adding a class **and** a wrapper here — an ambient gradient inlined
into a route is the drift this page exists to prevent.

## prefers-reduced-motion

One gate covers the whole layer: under `@media (prefers-reduced-motion: reduce)` all three stop
animating and keep their colour. The glow also settles at `saturate(0.5)` so it matches its resting
frame rather than jumping to full bloom.

The `.anim-*` utilities on the Motion page are **not** gated yet. That gap is theirs, not this
layer's.
